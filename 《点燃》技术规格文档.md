点燃 · 技术规格文档 V2.0
================

**文档类型：** MVP 完整技术规格，可直接用于 AI 辅助编程与后端实现  
**最后更新：** 2026-04-27  
**关联设计文档：** 点燃 PRD V4.0  
**AI 服务提供商：** MiniMax (MiniMax-Text-01)

1. 核心概念与术语表

-----------

> **核心隐喻：** 灵感是火种，行动是烈焰，脉络是草原。

| 术语      | 数据实体      | 关键特征                                        |
| ------- | --------- | ------------------------------------------- |
| **火种**  | `Spark`   | 灵感碎片，不可执行；可点燃成多团烈焰；逻辑删除；来源可追溯               |
| **烈焰**  | `Flame`   | 可执行探索任务；状态流转(burning→burned)；强制记录；取火次数上限10次 |
| **草原**  | `Prairie` | 跨时间叙事脉络容器；项目标题级；名称唯一；逻辑删除(存档)               |
| **野火**  | 虚拟概念      | `flames` 中 `prairieId IS NULL` 的集合          |
| **火盆**  | 页面视图      | 展示所有火种；底部导航 Tab 1                           |
| **探索页** | 页面视图      | 主动探索入口；底部导航中间按钮                             |
| **点燃**  | 动作        | 火种→多团烈焰，通过三步向导完成                            |
| **取火**  | 动作        | 烈焰完成→3粒新火种，用户筛选保留                           |
| **整理**  | 动作        | 野火分析→归入草原建议，用户逐条确认                          |

2. 数据模型 (IndexedDB)

-------------------

**数据库名称：** `kindling`  
**ORM/封装：** Dexie.js

### 2.1 `sparks` 表

| 字段                | 类型                                                           | 说明           |
| ----------------- | ------------------------------------------------------------ | ------------ |
| `id`              | string (UUID)                                                | 主键           |
| `content`         | string                                                       | 火种文本         |
| `sourceType`      | `'user'` \| `'ai_rekindle'` \| `'ai_seed'` \| `'ai_explore'` | 来源           |
| `sourceFlameId`   | string?                                                      | 取火来源烈焰       |
| `sourcePrairieId` | string?                                                      | 取火来源草原       |
| `sourceSparkId`   | string?                                                      | 点燃来源火种（用于追溯） |
| `createdAt`       | number (timestamp)                                           | 创建时间         |
| `isDeleted`       | boolean (默认 false)                                           | 逻辑删除         |

**查询火盆列表：**  
`sparks.where({ isDeleted: false }).reverse().sortBy('createdAt')`

### 2.2 `flames` 表

| 字段                     | 类型                        | 说明                  |
| ---------------------- | ------------------------- | ------------------- |
| `id`                   | string (UUID)             | 主键                  |
| `title`                | string                    | 烈焰标题                |
| `description`          | string                    | 详细描述（可选）            |
| `recommendationReason` | string                    | 推荐理由                |
| `searchPhrase`         | string                    | 探索口令                |
| `status`               | `'burning'` \| `'burned'` | 状态                  |
| `prairieId`            | string?                   | 所属草原ID（null为野火）     |
| `sourceSparkId`        | string?                   | 来源火种ID              |
| `igniteBatchId`        | string?                   | 点燃批次ID（同次点燃的多团烈焰共享） |
| `userRecord`           | string                    | 完成感想（强制，可为`''`）     |
| `completedAt`          | number?                   | 完成时间戳               |
| `createdAt`            | number                    | 创建时间                |
| `isDeleted`            | boolean (默认 false)        | 逻辑删除                |
| `rekindleCount`        | number (默认 0)             | 取火次数                |
| `lastRekindleTime`     | number?                   | 上次取火时间戳             |

**查询野火：**  
`flames.where({ prairieId: null, isDeleted: false }).reverse().sortBy('createdAt')`  
**查询草原下烈焰：**  
`flames.where({ prairieId: 'xxx', isDeleted: false }).reverse().sortBy('createdAt')`

### 2.3 `rekindleRecords` 表

| 字段                 | 类型            | 说明          |
| ------------------ | ------------- | ----------- |
| `id`               | string (UUID) | 主键          |
| `flameId`          | string        | 关联烈焰ID      |
| `sparksResult`     | string[]      | AI生成的3粒火种文本 |
| `retainedSparkIds` | string[]      | 用户保留的火种ID数组 |
| `createdAt`        | number        | 生成时间        |

### 2.4 `prairies` 表

| 字段            | 类型                         | 说明        |
| ------------- | -------------------------- | --------- |
| `id`          | string (UUID)              | 主键        |
| `name`        | string (unique)            | 草原名，项目标题级 |
| `description` | string                     | 描述（可选）    |
| `status`      | `'active'` \| `'archived'` | 活跃 / 存档   |
| `archivedAt`  | number?                    | 存档时间      |
| `createdAt`   | number                     | 创建时间      |

**唯一索引：** `name`  
**删除逻辑：** 用户点删除 → `status = 'archived'`。恢复 → `status = 'active'`。物理删除（二次确认）→ 删除记录，关联烈焰设置 `prairieId = null`。

### 2.5 `seedBuffer` 表

| 字段          | 类型                 | 说明    |
| ----------- | ------------------ | ----- |
| `id`        | string (UUID)      | 主键    |
| `content`   | string             | 火种文本  |
| `createdAt` | number             | 生成时间  |
| `used`      | boolean (默认 false) | 是否已展示 |

### 2.6 `deviceInfo` 表

| 字段             | 类型         | 说明     |
| -------------- | ---------- | ------ |
| `id`           | `"device"` | 固定主键   |
| `deviceUuid`   | string     | 设备唯一标识 |
| `lastSyncTime` | number?    | 上次备份时间 |

3. AI 服务接口规格

------------

**基础信息：**

* **AI 服务：** MiniMax-Text-01 (API 兼容 OpenAI Chat Completions)

* **代理层：** Cloudflare Workers

* **API 地址：** `https://api.minimax.chat/v1/text/chatcompletions_v2`

* **鉴权：** `Authorization: Bearer ${MINIMAX_API_KEY}`

* **通用配置：** `temperature: 0.7`, `max_tokens: 1000`

* **全局降级：** 接口不可用或返回非 JSON 时，前端展示备用 UI（手动输入），不阻塞流程。

### 3.1 点燃辅助

**端点：** `POST /api/ai/ignite`  
**描述：** 将灵感转化为多视角探索行动建议。

**输入规格：**

json

{
  "sparkContent": "用户想法文本",
  "existingPrairies": ["草原1", "草原2"]
}

**Prompt 指令：**

text

用户有一个想法或问题，想把它变成具体的探索行动。请按以下步骤帮助用户：
第一步：为这个想法生成3-5个不同的探索视角。视角类型可以包括：读一本书或文章、看一个视频或纪录片、写一篇自己的思考、找人讨论或采访、动手实践或体验。
第二步：为每个视角推荐一个具体的、低门槛的第一步行动建议。每个行动附上一个可在B站、知乎、小红书等平台搜索的自然语言短语。
第三步：汇总为一份行动草稿。
用户的想法：{sparkContent}
用户已有的草原：{existingPrairies}
返回格式（严格JSON，不要任何额外文字）：
{
  "perspectives": [
    {
      "type": "视角类型",
      "description": "视角描述",
      "firstStep": "第一步行动建议",
      "searchPhrase": "可搜索的自然语言短语"
    }
  ],
  "suggestedPrairie": "从已有草原中选最匹配的，或null",
  "newPrairieSuggestion": "若都不匹配，建议新草原名，或null"
}

**成功输出示例：**

json

{
  "perspectives": [
    {
      "type": "观看",
      "description": "通过短视频直观理解拉花手法",
      "firstStep": "看一个B站5分钟拉花入门教程",
      "searchPhrase": "咖啡拉花 新手入门教程"
    },
    {
      "type": "阅读",
      "description": "了解咖啡豆品种与风味的关系",
      "firstStep": "读《世界咖啡地图》第一章",
      "searchPhrase": "世界咖啡地图 在线阅读"
    }
  ],
  "suggestedPrairie": null,
  "newPrairieSuggestion": "咖啡探索"
}

**降级处理：** 接口失败 → 显示“AI辅助暂时不可用”，提供空白烈焰创建表单，用户手动输入标题、行动步骤。

### 3.2 取火

**端点：** `POST /api/ai/rekindle`  
**描述：** 完成探索后生成3个新灵感问题。

**输入规格：**

json

{
  "taskTitle": "任务标题",
  "taskDescription": "任务描述",
  "userRecord": "用户感想",
  "sourcePrairie": "所属草原"
}

**Prompt 指令：**

text

用户完成了一个探索任务并写下感想。请生成3个新问题或灵感，帮助继续探索。
任务标题：{taskTitle}
任务描述：{taskDescription}
用户感想：{userRecord}
所属草原：{sourcePrairie}
每个新问题/灵感应：

- 与探索内容相关
- 提供新视角，非总结
- 可进一步探索的方向（如读了书可推荐纪录片、播客等）
- 1-2句话，自然流畅
  返回严格 JSON：
  {
  "newQuestions": ["问题1", "问题2", "问题3"]
  }

**降级处理：** 失败时展示“AI暂时无法取火”，允许手动创建火种。

### 3.3 整理

**端点：** `POST /api/ai/organize`  
**描述：** 未归类任务智能归类建议。

**输入规格：**

json

{
  "unclassifiedTasks": [{"title": "读《局外人》"}],
  "existingPrairies": ["存在主义探索"]
}

**Prompt 指令：**

text

用户有一些未分类任务和已有草原。请分析给出归类建议。
未分类任务：{逐一列出title}
已有草原：{existingPrairies}
规则：

- 高度相关的可归入已有草原
- 多任务围绕一个具体新方向（如"存在主义文学探索"）可建议创建新草原
- 不确定的保持沉默
- 同一任务不出现在两个建议中
  返回严格 JSON：
  {
  "suggestions": [
    {"action": "merge", "taskIndices": [0], "targetPrairie": "草原名", "reason": "理由"},
    {"action": "create", "taskIndices": [1,2], "newPrairieName": "新草原名", "reason": "理由"}
  ]
  }

### 3.4 探索生成

**端点：** `POST /api/ai/explore`  
**描述：** 根据关键词生成多角度火种。

**输入规格：**

json

{
  "query": "关键词或问题",
  "existingPrairies": ["草原1"]
}

**Prompt 指令：**

text

用户对某个主题感兴趣。请围绕这个主题生成5-8粒火种（灵感/问题/方向）。
用户输入：{query}
已有草原：{existingPrairies}
每粒火种应：

- 从不同角度切入（历史/科学/文化/实践等）
- 标注视角类型（阅读/观看/实践/讨论/思考）
- 1-2句话，引发好奇心
  返回严格 JSON：
  {
  "sparks": [
    {"content": "火种文本", "type": "视角类型"}
  ],
  "suggestedPrairie": "匹配的已有草原或null",
  "newPrairieSuggestion": "新草原建议或null"
  }

### 3.5 种子缓存补充

**端点：** `POST /api/ai/seed-buffer`  
**描述：** 补充火种缓存池（后台自动调用）。

**输入规格：**

json

{
  "recentSparks": ["近期火种样本"],
  "existingPrairies": ["草原1"]
}

**Prompt 指令：**

text

生成5个有趣的知识探索问题，作为灵感来源。
用户近期关注：{recentSparks}
已有草原：{existingPrairies}
每个问题应：

- 指向具体可探索的知识缺口
- 涵盖不同领域
- 1-2句话
  返回严格 JSON：
  {
  "questions": ["问题1", ...]
  }

### 3.6 草原重名校验

**端点：** `POST /api/validate-prairie`  
**输入：** `{ deviceUuid, prairieName }`  
**逻辑：** 读取 OSS 备份文件中的 `prairies` 数组检查重名。  
**输出：** `{ exists: boolean, message?: string }`  
**降级：** OSS 不可用时返回 `exists: false`，由前端本地校验兜底。

4. 核心业务规则与交互

------------

### 4.1 点燃三步向导

**页面路径：** `/hearth/kindle/:sparkId`

1. **视角选择页**
   
   * 调用 `POST /api/ai/ignite` 获取 `perspectives`。
   
   * 展示视角卡片列表，每张含类型、描述、第一步建议预览。
   
   * 用户可多选视角（至少选1个，或点击“跳过，直接创建”进入手动模式）。

2. **行动确认页**
   
   * 为选中的每个视角显示推荐的 `firstStep` 和 `searchPhrase`。
   
   * 用户可编辑每一步的行动描述和搜索口令。
   
   * 用户可删除某个视角对应的烈焰。

3. **最终确认页**
   
   * 展示即将创建的烈焰卡片预览（标题由AI自动生成或用户手动输入）。
   
   * **草原选择器**：带搜索的输入框，显示匹配的已有草原；输入新名称则保存时自动创建新草原。
   
   * 提交时校验草原名唯一（前端+远端）
   
   * 确认保存：批量创建烈焰记录（`igniteBatchId` 为同一UUID，`sourceSparkId` 为当前火种ID），火种保持不变。
   
   * 弹出平台跳转面板，搜索口令已复制到剪贴板。

### 4.2 完成燃烧与取火

1. 烈焰详情页点击“完成燃烧” → `/prairie/flame/:id/complete`

2. 感想输入框（`<textarea>`），**必填**，可为空格。

3. 提交时：
   
   * 更新烈焰：`status='burned'`, `userRecord`, `completedAt=now`
   
   * 取火次数与冷却检查（`rekindleCount >= 10` 禁止；`Date.now() - lastRekindleTime < 5000` 按钮置灰并倒计时）
   
   * 跳转到取火页 `/prairie/flame/:id/rekindle`

4. 取火页：
   
   * 上方：手动火种输入框，实时写入 `sparks` 表（`sourceType='user'`, 可关联来源）
   
   * 下方：调用 `POST /api/ai/rekindle`，展示“AI正在帮你取火中...”，返回后显示3粒火种卡片
   
   * 每粒火种有【保留】【丢弃】按钮；保留后立即入库（`sourceType='ai_rekindle'`, `sourceFlameId`, `sourcePrairieId`）
   
   * 【重新取火】按钮，同冷却和次数限制
   
   * 退出页面：已保留的火种已入库，未操作的AI结果不保存
   
   * 历史取火记录可展开，不可重新操作
   
   * 若感想为空/空格，烈焰详情展示**“默默燃尽”**

### 4.3 整理流程

1. 草原 Tab 内点击“整理” → `/prairie/organize`

2. 收集 `prairieId === null AND isDeleted === false` 的所有烈焰标题

3. 调用 `POST /api/ai/organize`，展示**分组总览**卡片列表：`merge` 和 `create` 建议

4. 点击某个总览项进入**逐条确认**：显示受影响的烈焰标题和理由
   
   * 【确认归入】【跳过】【手动调整】

5. 确认后更新 `flames.prairieId` 或创建新草原再关联

6. 用户可随时退出，已确认生效，未确认保留

7. **触发提示：** 用户累计完成 ≥3 次取火后，野火区底部展示轻量提示入口

### 4.4 探索页

* 独立导航页，输入框占位：“输入关键词或问题开始探索”

* **有输入时**：调用 `/api/ai/explore`，展示多角度火种流，每粒火种可“加入火盆”或直接“点燃”

* **无输入时**：展示推荐：
  
  * 基于已有草原的延伸方向（随机展示）
  
  * 跨领域随机组合示例（静态或简单规则）
  
  * “换一批”刷新推荐

### 4.5 火种动态与缓存池

* 初始预置 5 粒 `ai_seed` 火种

* `seedBuffer` 表维护 ≥10 粒未使用火种

* 后台在 `used=false` 数量 < 10 时，调用 `/api/ai/seed-buffer` 补充 5 粒

* 用户点击火盆顶部“换一批”：
  
  1. 从 `seedBuffer` 取 5 粒 `used=false`，标记为 `used=true`
  
  2. 批量插入 `sparks` 表（`sourceType='ai_seed'`）
  
  3. 当前火盆列表立即刷新显示新火种

### 4.6 草原生命周期

* **创建：** 手动创建（输入唯一名称），或整理时AI建议创建

* **编辑：** 名称修改需校验唯一性

* **存档（删除）：** 弹窗确认 → `status='archived'`，烈焰保持关联

* **恢复：** 存档区点击恢复 → `status='active'`

* **彻底删除：** 存档草原详情页内二次确认 → 物理删除记录，关联烈焰 `prairieId=null`

### 4.7 火种/烈焰删除

* 火种：`isDeleted=true`，不影响已点燃烈焰

* 烈焰：`isDeleted=true`，不影响已取火产生的火种

### 4.8 数据备份与还原

* **备份：** `/settings` 手动点击 → 序列化 IndexedDB 全表 → POST Worker 上传至 OSS `devices/{uuid}/backup.json`，同时备份至 `backups/backup_{timestamp}.json`

* **还原：** 手动点击 → 下载 OSS backup.json → 用户确认 → **清空本地数据库** → 导入

* **无自动备份**
5. 前端架构设计

---------

### 5.1 技术栈

* **核心框架：** React 18 + TypeScript

* **样式：** Tailwind CSS

* **状态管理：** Zustand 或 React Context（局部状态为主）

### 5.4 组件设计要点

* **虚拟滚动：** 火盆和野火列表使用 `react-window` 或 `@tanstack/virtual`。

* **平台跳转面板：** 一个 Modal，展示图标列表，点击拼接 URL 跳转。

* **草原选择器：** 可复用的 `Combobox` 组件，支持异步搜索本地草原集合，允许输入新名称。

* **取火页状态：** 使用 `useReducer` 管理 AI 结果、手动输入、保留状态。

* **响应式：** 移动优先，最大宽度 `max-w-lg` 居中。

### 5.5 状态同步策略

* 各页面 `useEffect` 依赖路由参数，挂载时从 Dexie 查询最新数据。

* 底部导航切换时组件重新挂载并拉取数据。

* 不采用全局实时推送。

7. 错误处理与边缘情况

------------

| 场景             | 处理策略                          |
| -------------- | ----------------------------- |
| IndexedDB 不可用  | 提示“存储不可用，请检查浏览器权限”，阻止进一步交互    |
| AI 接口超时 (>10s) | AbortController 取消，降级为手动模式 UI |
| AI 返回非 JSON    | `tryParse` 失败则降级，记录日志但不中断用户操作 |
| 取火冷却期点击        | 按钮置灰 + 倒计时文字（`重新取火 (5s)`）     |
| 取火次数达上限        | 按钮永久置灰，提示“已达上限”               |
| 草原名重名        | 前端提示 + 后端 409，阻止创建            |
| OSS 备份失败       | 提示“网络异常，稍后重试”                 |
| 还原文件损坏         | 校验 JSON 顶层键，失败则拒绝还原           |
| 设备 UUID 丢失     | 视为全新设备，旧备份无法恢复（已知限制）          |
| 火种缓存池空空        | “换一批”按钮置灰，提示“稍后试试”，后台立即尝试补充   |
| 点燃视角选择为空       | 至少选1个视角，否则禁用下一步               |

### 9.3 MiniMax Worker 示例代码

javascript

export default {
  async fetch(request, env) {
    const { pathname } = new URL(request.url);
    if (pathname.startsWith('/api/ai/')) {
      const body = await request.json();
      const prompt = buildPromptByEndpoint(pathname, body);
      const response = await fetch('https://api.minimax.chat/v1/text/chatcompletions_v2', {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${env.MINIMAX_KEY}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          model: 'MiniMax-Text-01',
          messages: [{ role: 'user', content: prompt }],
          temperature: 0.7,
          max_tokens: 1000
        })
      });
      const data = await response.json();
      const content = data.choices[0].message.content.replace(/```json|```/g, '').trim();
      return new Response(content, { headers: { 'Content-Type': 'application/json' } });
    }
    // 其他路由...
  }
}
