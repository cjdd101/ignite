二、"标签"与"草原"概念不一致问题汇总 需修改

设计文档明确：草原 = 项目标题级叙事脉络，标签在设计文档中应指代草原。
但技术规格中大量使用"标签"概念，与设计文档不一致。

三、其他不一致问题

3.1 igniteBatchId 必填性

技术规格：igniteBatchId: string（必填，无?）

设计文档：未明确说明，但"手动创建烈焰"场景未提及此字段

建议：技术规格改为 igniteBatchId?: string，手动创建时可不填

3.2 取火冷却时间示例矛盾

文档描述：冷却时间 ≥ 5秒

错误提示示例：重新取火 (3s)

建议：统一为5秒，错误提示改为 重新取火 (5s)

3.3 草原名称字段歧义

问题：AI接口中用"标签名"指代草原名，但 prairies 表的 name 字段就是草原名

混淆场景：

  • 整理接口：targetTag: "已有标签名" → 实际指 prairie.name
  • 点燃接口：suggestedTag → 实际建议的是 prairie.name

建议：统一使用"草原"术语，避免歧义
